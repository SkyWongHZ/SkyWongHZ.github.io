移动端页面自适应方案小结

发表于 2016-10-13

手机端页面自适应
最近这两天在做手机端页面的时候,一直碰到纠结的事，在自己的手机上看没有样式bug，但是在别人的手机页面上看的时候却有样式的bug，说到底是对原先样式布局的方式没有一个系统的总结,一味的在为了写页面而写页面,没有去详细了解过为什么要用这种方式来实现,移动端页面自适应方案rem布局方案,以后做的话实现过程如下:

1和设计师商量好设计稿尺寸,一般是750px(iPhone6)尺寸,也可以是640px尺寸;
2实际在量取页面过程中用375px去量取;
3确定iphone6的根元素字体大小，在iphone6上一般设置根元素字体大小为16px 或100px；
4动态js添加不同屏幕下的根元素尺寸大小
document.getElementsByTagName(‘html’)[0].style.fontSize = window.screen.width/23.4375+’px’;
上面这段代码为设置根元素字体大小为16px，
如果是根元素大小是100px，即把23.4375改成3.75即可;

注意:根元素字体大小不能设置为1px或者10px,会出问题,用16px来计算的话,对于不支持字体大小的css样式也没有问题,用100px来设置的话可能在一些css上会有问题.

(刚才亲测用100px的方式原先页面的字体大小过大,所以还是用设置根元素字体大小为16px的大小最合适.)

5针对个别手机型号的宽度添加特定的根元素尺寸大小,而且对一些特定手机样式上的css做媒体查询来适配页面.

css媒体查询相关知识点:

1-1屏幕尺寸相关:
超小屏幕 手机 (<768px)
小屏幕 平板 (≥768px)
中等屏幕 桌面显示器 (≥992px)
大屏幕 大桌面显示器 (≥1200px)
1-2媒体查询的具体用法：
@media (max-width: @screen-xs-max) { … }
@media (min-width: @screen-sm-min) and (max-width: @screen-sm-max) { … }
@media (min-width: @screen-md-min) and (max-width: @screen-md-max) { … }
@media (min-width: @screen-lg-min) { … }
