git指令学习

发表于 2017-01-02

cd 进入下一层级目录

pwd 显示当前目录
ls-ah 查看git文件下的隐藏目录
git add 提交文件
git add . 提交所有文件

拿到一个项目前:
1.初始化仓库 git init
2.提交文件到git仓库共两步骤
2.1 git add;
2.2 git commit
PS：提交全部文件的时候用”git add .”

git diff <文件名> 查看文件变化
git status 查看文件每时每刻的状态

git log 查看历史版本记录
只想查看内容变化，只要加上– pretty=oneline

git reset –hard <版本号>
恢复到指定时间内提交的版本号 版本号只要去github里面寻找即可.

git reflog 查看每一次操作指令记录

git rm <文件名> 删除一个文件

关联一个远程库，使用命令git remote add origin https://github.com/michaelliao/gitskills.git

git push origin master 把文件推送到远程服务器上

git push -f origin master 第一次推送文件的时候

算是第二次理解了：

其实之前反反复复的看git相关的教程已经有一段时间了,git的操作也总是云里雾里,实际上并没有一个深入的理解.

git的操作现在就记住以下几点:

1.把文件克隆下来
git clone

2同步自己的更新
git pull
git add * /rm
git commit -m “add”
git push

PS:可以把以上命令打包在一个.bat文件里,每次需要执行的时候只要在相关目录下git xx.bat就行了。

如果你实在不知道发生了什么，你又确定你自己的本地目录是最新的。你可以把本地的.git删除，然后新建一个仓库，git remote add ……然后使用上面的无脑四重奏，重新push上去。

如果你确定网上的是最新的，你本地被你毁的面目全非了，也不要怕，把这个本地目录整个删除，重新git clone就好了。
