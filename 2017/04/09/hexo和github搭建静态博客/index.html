hexo和github搭建静态博客

发表于 2017-04-09 | 更新于 2018-05-26

以前一直断断续续的想整自己的博客，一直没有花心思在上面，但是现在我下定决心要走互联网技术方向了,凭什么我不能走的比别人更快，自认为学习能力还可以,凭什么我就不能弯道超车,你要明白你不是一头骡子，不要逆来顺受，要学会思考,在公司的确写业务代码为重，相应的你要思考在写框架,写业务代码的时候你有没有去深入理解思考本质的东西,否则你只是一头骡子，只会随着年纪的增长,对熟悉的那一部分技术会使用，但是无法成为软件工程师,只是一个纯码农,要防止自己处在”温室”里面,你要不断的给自己施加压力，逼迫自己去成长.

配置博客的思路就是基于 hexo+github配置静态博客的方式,相应的可以参考的配置教程如下:
http://hifor.net/2015/07/01/%E9%9B%B6%E5%9F%BA%E7%A1%80%E5%85%8D%E8%B4%B9%E6%90%AD%E5%BB%BA%E4%B8%AA%E4%BA%BA%E5%8D%9A%E5%AE%A2-hexo-github/

在它的教程基础上我配置静态博客的时候要注意以下几点:
1端口号4000可能被占用,可以在github中临时改为其他端口 hexo s -p 端口号;
2需要配置ssh key;
3配置blog/_config.yml文件时中间的空格符号千万不能少(这坑千万要注意);
4 blog/_config.yml文件下deplo应该配置成以下形式:
deploy:
type: git
repo: git@github.com:username/username.github.io.git
而不应该照着教程上的方法配置，需要改成以上的代码片段
5重装系统后要考虑一个问题，怎么不重新部署一遍hexo，每次重装系统,每次在部署一遍不觉得很麻烦吗?这次重装hexo后完全按教程部署了一遍后,hexo d -g在github上明明有,但是页面上显示不出来,因为你换了config文件下的theme(主题)，被这问题折腾了这么久，sigh!
