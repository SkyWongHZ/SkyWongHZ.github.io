ionic混合开发第四周工作小结(4.17-22)

发表于 2017-04-20

这周汇出行项目做了扫码解锁bug修复,信用分积分功能,学会了angualr路由传参,本地存储localstorage相关,刚礼拜天下午看了一下午的谷歌开发者工具,2017年前端学习有用http://mp.weixin.qq.com/s/8vz1aIeVpRHmU2E2-7zzsQ

ionic 环境配置
17年4月18号早上折磨了一早上,卸载系统并重装,到最后再重装ionic环境,一直安装不上去,最后问题出在公司网速上,下次要安装东西尽量在下班没人或自己家里安装(java home环境是8) http://www.jianshu.com/p/b9757a5bcb07

chrome dev tool
1对chrome dev tool有了基本基本的了解,下次想深入了解具体的一块功能的话可以参考这篇文章：http://wiki.jikexueyuan.com/project/chrome-devtools/;
2信用积分相关的功能模块和之前做的我的行程,我的消息几乎都是差不多,按照个人的理解,静态数据就存储在本地,动态数据就从服务器(后台人杰那边拿),理解了cookie与webStorage(h5)新特性的区别,cookie一般是用来存储小容量数据,localStoragr上存储的数据永久保存,sessionStorage上只是临时保存数据;
3扫码解锁部分自己没去看仔细，没把jsonp请求放在cordova自带的扫码插件程序里面,编程前一定要考虑清楚问题的本质,像扫码解锁部分就应该(1)判断state有无来选择进入登录页面还是其他页面；(2)根据后台给出的state状态0,1,2分别进入充值页面,实名认证页面,登录成功下扫码解锁;
4认识到一点，现阶段自己的能力还不足以去和为了面子逞一时之快,细水长流,三十年河东,三十年河西,现阶段就努力拼命提升自己,被骂也要撑下去!
